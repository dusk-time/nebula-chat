## 📝 测试步骤

### 方法 1: 使用全局测试函数（最快）

1. **启动应用**
```bash
cd F:\nebula-chat
npm run dev
```

2. **打开浏览器控制台**（F12）

3. **运行测试命令**：
```javascript
// 测试消息显示
window.__testMessage('第一条测试消息');

// 再发一条
window.__testMessage('第二条测试消息 ' + new Date().toLocaleTimeString());
```

4. **检查结果**：
   - 应该看到 "🧪 测试消息已添加" 日志
   - 聊天界面应该显示测试消息
   - 按 **L 键** 查看调试日志

### 方法 2: 使用调试面板

1. 启动应用后，点击右上角的 **🐛 调试** 按钮
2. 点击 **🧪 测试消息** 按钮
3. 检查消息是否显示

## ❌ 如果还是不行

### 检查 1: 前端是否收到事件

在浏览器控制台运行：
```javascript
// 手动触发事件
window.dispatchEvent(new CustomEvent('p2p-message-received', {
  detail: {
    from_peer_id: 'test',
    from_name: 'Test',
    content: '手动测试',
    timestamp: Date.now() / 1000,
    message_id: 'manual-test'
  }
}));
```

**如果消息显示了** → 事件监听正常，问题在后端事件推送

**如果消息没显示** → 前端事件监听有问题

### 检查 2: 后端是否发送事件

查看后端终端日志，应该看到：
```
📬 [后端] 转发收到消息事件到前端 from=...
✅ [后端] 事件发送成功
```

**如果没有看到** → 后端 P2P 引擎没收到消息

**如果看到但前端没收到** → Tauri 事件系统问题

### 检查 3: 连接状态

点击 **🐛 调试** → **🔄 刷新连接**

- 如果连接数为 0 → P2P 连接没建立
- 如果连接数为 1 → 连接正常
